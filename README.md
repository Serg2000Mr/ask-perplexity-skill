# Скилл ask-perplexity для вашей IDE

Скилл для получения консолидированных ответов из множества веб-источников через [Perplexity AI](https://www.perplexity.ai/) прямо из чата с AI-агентом.

Поддерживает **Claude Code** (VSCode extension, CLI) и **Cursor**. Для других IDE запрашивайте адаптацию у агента.

## Что это

Добавляет команду `/ask-perplexity` в ваш AI-агент. Вопрос отправляется в Perplexity API, ответ выводится прямо в чат. Исключение: `sonar-deep-research` сохраняет результат в файл (объём может превышать 30 КБ), агент читает файл и даёт краткое резюме. Полезно когда нужно:

- Провести ревью плана или архитектурного решения с внешней точкой зрения
- Получить консолидированный ответ из десятков источников — в отличие от обычного поиска, Perplexity сразу даёт осмысленный вывод со ссылками
- Уточнить незнакомый API, библиотеку или формат
- Проверить поведение платформы или граничные случаи перед принятием решения

## ⚠️ Галлюцинации — проверяйте каждое фактическое утверждение

Perplexity собирает ответ из источников и LLM поверх них и регулярно галлюцинирует. Типичные сбои:

- **Придуманные CVE-номера, названия продуктов, версии, заголовки статей** — выглядят правдоподобно, но не существуют
- **Неправильная атрибуция цитат** — реальное утверждение приписано не тому источнику
- **Закрытые уязвимости как актуальные** — пофикшенный баг описан как всё ещё опасный
- **Неверные API, флаги, коды выхода** — «используй exit 2 чтобы заблокировать», когда документация говорит «exit 0 + JSON»
- **Гибриды** — два реальных факта склеены в утверждение, ложное в целом

**Правила работы:**

1. Любое фактическое утверждение — гипотеза, не истина. Особенно если от него зависят план, код, архитектурное решение
2. Проверять любой именованный объект до действия: CVE — в NVD или у первоисточника, продукт — на сайте вендора, API — в официальной документации, версия — в changelog
3. Первоисточники из блока `Sources:` важнее пересказа Perplexity. Если Perplexity цитирует документ — открыть документ и перечитать
4. Не копировать утверждения в код, план или отчёт без независимого подтверждения
5. Если критичное утверждение нельзя проверить — либо явно помечать как неподтверждённое при передаче, либо отбрасывать

Perplexity полезен как внешний синтезатор и указатель на источники. Его резюме — не доказательство.

## Получение API-ключа и пополнение баланса

Perplexity имеет два отдельных баланса: для веб-сайта и для API. Подписка Pro и баланс веб-интерфейса **не дают** доступа к API — нужно отдельно пополнить баланс API-платформы.

### Шаг 1. Создайте API-ключ

1. В [perplexity.ai](https://www.perplexity.ai/) откройте **Аккаунт → API-платформа**

   <img width="571" height="656" alt="Меню аккаунта Perplexity" src="https://github.com/user-attachments/assets/07bc8279-82e4-4562-ba1b-a2d1e0f68eb6" />

2. Создайте ключ API (начинается с `pplx-`) и скопируйте его

### Шаг 2. Пополните баланс API

Перейдите в [API-платформу](https://www.perplexity.ai/settings/api) и пополните баланс. Оплата по факту потребления, за токены. Актуальные тарифы: [docs.perplexity.ai](https://docs.perplexity.ai/).

### Шаг 3. Добавьте ключ в настройки вашей IDE

Инструкции — в разделе [Установка](#установка) ниже.

## Стоимость и лимиты API

Оплата по факту потребления (за токены). Актуальные тарифы: [docs.perplexity.ai](https://docs.perplexity.ai/).

## Модели Perplexity API (март 2026)

| Model ID | Контекст | Что делает | Когда полезна |
|----------|----------|------------|---------------|
| `sonar` | 128k | Быстрый веб-поиск, Q&A, суммаризация | Короткие фактические вопросы, уточнения API |
| `sonar-pro` | 200k | Глубокий поиск с расширенным контекстом | Анализ, ревью, исследование |
| `sonar-reasoning-pro` | 128k | Многошаговое рассуждение, chain-of-thought | Сложный анализ, структурированные выводы |
| `sonar-deep-research` | 128k | Синтез из сотен источников | Исчерпывающий документ по теме |

Список моделей может измениться. Актуальный перечень: [docs.perplexity.ai](https://docs.perplexity.ai/).

### Как агент выбирает модель

Агент (Claude Code и Cursor) выбирает модель автоматически — не нужно указывать её вручную.

| Тип задачи | Модель | Что получишь |
|------------|--------|--------------|
| Быстрый факт, уточнение API, короткий вопрос | `sonar` | Краткий точный ответ |
| Ревью плана, архитектурное решение, исследование вопроса | `sonar-pro` | Развёрнутый анализ с источниками |
| Многошаговое рассуждение, сложный анализ | `sonar-reasoning-pro` | Структурированный вывод с цепочкой аргументов |
| Нужен полный отчёт по теме из сотен источников | `sonar-deep-research` | Большой итоговый документ (сохраняется в файл) |

Результаты тестирования на вопросе «Какая последняя версия .NET SDK вышла в 2025 году?» показали ожидаемое поведение: `sonar` ответил коротко и точно (.NET 10.0.1, декабрь 2025), `sonar-pro` дал более развёрнутый ответ с контекстом по версиям, `sonar-reasoning-pro` выдал структурированный вывод с пошаговым рассуждением, а `sonar-deep-research` сгенерировал аналитический отчёт объёмом больше 30 КБ — полезно для глубоких исследований, но избыточно для фактических вопросов.

## Troubleshooting

| Проблема | Решение |
|----------|---------|
| `PERPLEXITY_API_KEY not set` | Настройте ключ (см. инструкцию для вашей IDE) |
| HTTP 401 | Проверьте что ключ начинается с `pplx-` и не истёк |
| HTTP 429 | Превышен лимит запросов, подождите 60 секунд |
| `python3: command not found` | Установите Python 3 |
| Пустой ответ | Проверьте интернет-соединение |
| `curl: (35)` / `HTTP 000` на Windows | schannel не может проверить CRL/OCSP. В скрипте уже есть `--ssl-no-revoke`. Если проблема всё равно есть — обновите curl или проверьте корпоративный MITM-proxy |
